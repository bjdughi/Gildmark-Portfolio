# Purpose

This document defines the professional capability claims supported by the Gildmark portfolio.

Capability claims describe the architectural, engineering, and leadership competencies demonstrated through the project's evidence.

The purpose of these claims is not self-promotion.

Their purpose is to provide reviewers with explicit hypotheses that can be evaluated against the supporting evidence presented throughout the portfolio.

---

# Capability Philosophy

Professional capability should be demonstrated rather than asserted.

The portfolio therefore separates:

* capability claims
* supporting evidence
* reviewer conclusions

Readers are encouraged to evaluate the evidence independently.

Capability claims simply identify the competencies the portfolio intends to demonstrate.

---

# Traceability

Each capability claim should be traceable to representative evidence.

Representative evidence should, in turn, trace to:

* architectural concepts
* architectural principles
* architectural decisions
* implementation artifacts

This traceability enables reviewers to evaluate each claim objectively.

---

# Evaluation

The portfolio intentionally avoids assigning proficiency ratings or subjective assessments.

Its objective is to present sufficient evidence for experienced reviewers to reach their own conclusions regarding demonstrated capability.

Capability claims therefore function as evaluation guides rather than conclusions.

---

# Scope

This document identifies the professional capabilities supported by the portfolio.

It does not replace the underlying evidence.

Individual artifacts remain the authoritative source for evaluating each claim.
