# Capability Matrix

## Purpose

This document maps professional capabilities to the evidence contained within the Gildmark portfolio.

Unlike the Evidence Register, which catalogs individual artifacts, the Capability Matrix organizes those artifacts by the capabilities they demonstrate.

Its purpose is to provide multiple perspectives over the same body of evidence.

Depending on context, the same capability may support a résumé bullet, an interview response, an architectural essay, a consulting discussion, or an executive briefing.

Every capability listed below should be supported by one or more concrete artifacts contained within the Evidence Register.

# Portfolio Coverage

The Capability Matrix provides a high-level assessment of portfolio coverage.

Capabilities assessed as **Exceptional** are considered distinguishing strengths and should appear prominently in résumé material and interview discussions.

Capabilities assessed as **Strong** are well supported but may benefit from additional narrative or representative examples.

Capabilities assessed as **Developing** indicate areas where additional portfolio artifacts are planned to improve communication rather than technical depth.

As the portfolio evolves, this matrix should be reviewed whenever a significant new artifact is added or an existing capability is strengthened.

---

# Governance

The Capability Matrix is a derived document.

It should never become the source of truth.

New artifacts should first be evaluated and entered into the Evidence Register.

Only after evidence has been cataloged should this matrix be updated to reflect the additional capabilities that the evidence supports.

This preserves traceability between claims and supporting artifacts while ensuring that every capability presented by the portfolio remains grounded in demonstrable evidence.
